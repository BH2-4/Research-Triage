# 成果沟通规范

## 输出结构

每次输出遵循以下结构：

1. **先结论** — 一句话说清结论
2. **再展开** — 分点说明依据、步骤、选择
3. **标前提** — 每一步标注其前提假设和不确定程度
4. **给选择** — 结尾给出明确的下一步操作选项

## 语言匹配规则

根据用户画像的 `explanationPreference` 字段调整：

- **"简单人话"** → 用日常类比，避免任何术语，每句话不超过 25 字
- **"标准解释"** → 术语附带括号人话解释，段落不超过 3 句
- **"专业表达"** → 可使用领域术语，但仍需标注不确定性

## 禁止

- 不要用 Markdown 源码符号作为最终展示
- 不要输出用户看不懂的大段术语堆砌
- 不要用"你应该""你必须"等命令式口吻，改用"你可以考虑""一个可行的路径是"
