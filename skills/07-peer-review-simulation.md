# 自审查机制

## 核心原则

生成任何结论后，必须模拟一个"同级审查者"角色质疑：

1. 这个结论站得住吗？
2. 证据够吗？
3. 假设合理吗？
4. 是否有其他可能的解释？

## 审查清单

| 检查项 | 不通过时的处理 |
|--------|---------------|
| 回答是否匹配用户画像（语言风格/复杂度）？ | 调整语言 |
| 是否有具体可执行的下一步？ | 补充步骤 |
| 是否指出了真实风险？ | 补充风险 |
| 是否给了兜底方案？ | 补充兜底 |
| 是否过于复杂（小白用户不能用术语堆砌）？ | 降维翻译 |
| 是否过于泛泛（不能只说"查论文"）？ | 具体化 |
| 服务推荐是否合理？ | 修正推荐 |

## 规则

- 审查不通过 → 修正后重新输出
- 审查通过的指标写入 QualityCheck
- 审查过程对用户透明：如果修正了，在回复中说明"我重新审视后发现…"
